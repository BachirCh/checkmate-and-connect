  <action>
Create secure webhook endpoint that validates Sanity webhook signatures and triggers on-demand revalidation for blog pages.

**app/api/revalidate/route.ts:**
- Import `revalidatePath` from `next/cache`
- Import `NextRequest, NextResponse` from `next/server`
- Import `parseBody` from `next-sanity/webhook`

**Export async POST function:**
- Accept `req: NextRequest`
- Use try/catch for error handling

**Inside POST handler:**
1. **Parse and validate webhook:**
   - Call `parseBody(req, process.env.SANITY_WEBHOOK_SECRET)`
   - Destructure: `{ isValidSignature, body }`
   - If `!isValidSignature`, return 401 response: `{ message: 'Invalid signature' }`

2. **Check document type:**
   - Verify `body._type === 'blogPost'`
   - If not blogPost, return early with `{ message: 'No revalidation needed' }`

3. **Revalidate blog pages:**
   - Always revalidate listing: `await revalidatePath('/blog')`
   - If `body.slug?.current` exists, revalidate detail page: `await revalidatePath(\`/blog/\${body.slug.current}\`)`
   - Return success: `{ revalidated: true, now: Date.now() }`

4. **Error handling:**
   - Catch any errors
   - Log to console: `console.error('Webhook error:', err)`
   - Return 500: `{ message: 'Error processing webhook' }`

**Security notes:**
- SANITY_WEBHOOK_SECRET must be set in environment variables
- `parseBody` validates HMAC signature using webhook secret
- Invalid signatures rejected before any revalidation occurs
- Per RESEARCH.md Pattern 5, this follows official Sanity webhook security pattern

**.env.local.example:**
- Add commented line: `# SANITY_WEBHOOK_SECRET=your-webhook-secret-from-sanity`
- Add explanation comment above: `# Webhook secret for Sanity on-demand revalidation. Generate in Sanity Manage â†’ Project â†’ API â†’ Webhooks`

IMPORTANT: This webhook is optional for Phase 7 MVP but recommended for production. Without it, published blog posts may not appear for minutes/hours until ISR naturally rebuilds. With webhook, posts appear within seconds.
