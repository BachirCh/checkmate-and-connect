<task type="auto">
  <name>Task 2: Add JSON-LD structured data for blog posts</name>
  <files>
    app/(public)/blog/[slug]/page.tsx
  </files>
  <action>
Add Article schema JSON-LD structured data to blog post pages for rich search results in Google.

**Update app/(public)/blog/[slug]/page.tsx default export:**
- After fetching post data, before return statement, create jsonLd object
- Follow JSON-LD Article schema from RESEARCH.md Pattern 4

**JSON-LD object structure:**
```typescript
const jsonLd = {
  '@context': 'https://schema.org',
  '@type': 'BlogPosting',
  headline: post.title,
  description: post.excerpt || '',
  image: post.coverImage
    ? urlFor(post.coverImage).width(1200).height(630).url()
    : undefined,
  datePublished: post.publishedAt,
  author: {
    '@type': 'Person',
    name: post.author,
  },
  publisher: {
    '@type': 'Organization',
    name: 'Checkmate & Connect',
    url: process.env.NEXT_PUBLIC_SITE_URL || 'https://example.com',
  },
};
```

**Render JSON-LD in page:**
- Add script tag in JSX return before main element
- Use `<script type="application/ld+json" dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }} />`
- This is safe in Server Components (no hydration duplication issue per RESEARCH.md Pitfall 7)

**Why BlogPosting not Article:**
- `BlogPosting` is more specific schema type for blog content
- Google's Rich Results Test prefers BlogPosting for blog posts
- Article is generic, BlogPosting signals it's a blog

**Required fields for Google Rich Results:**
- headline (title)
- image (cover image with 1200x630 dimensions)
- datePublished (publishedAt)
- author (Person schema)

IMPORTANT: Server Components in Next.js 16 handle JSON-LD without duplication. Do NOT use Client Component for JSON-LD rendering.
