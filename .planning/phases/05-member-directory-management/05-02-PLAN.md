  <action>
Create Server Actions for admin member management operations with authorization and revalidation.

**File structure:**
```typescript
'use server'

import 'server-only' // CRITICAL: prevent client-side bundle inclusion
import { revalidatePath } from 'next/cache'
import { verifySession } from '@/lib/auth/dal'
import { writeClient } from '@/lib/sanity/write-client'
```

**Implement four actions:**

**1. approveMember(memberId: string)**
- Call `await verifySession()` first (throws/redirects if not admin)
- Use `writeClient.patch(memberId).set({ status: 'approved', approvedAt: new Date().toISOString() }).commit()`
- Revalidate BOTH paths: `revalidatePath('/admin/members')` AND `revalidatePath('/members')`
  - Admin path: updates admin table immediately
  - Public path: approved member appears in directory immediately (no cache staleness)
- Return `{ success: true }` on success
- Wrap in try/catch, return `{ success: false, error: 'Failed to approve member' }` on error
- Log errors with console.error for debugging

**2. rejectMember(memberId: string, reason?: string)**
- Similar structure to approveMember
- Patch: `{ status: 'rejected', rejectedAt: new Date().toISOString(), rejectionReason: reason || 'Did not meet approval criteria' }`
- Only revalidate `/admin/members` (rejected members never appear publicly)
- Return structured result

**3. deleteMember(memberId: string)**
- Use `writeClient.delete(memberId)` for permanent deletion
- Revalidate BOTH paths (if deleted member was approved, remove from public directory)
- Return structured result

**4. editMember(memberId: string, updates: { name?: string; jobTitle?: string; company?: string; linkedIn?: string })**
- Accept partial updates object
- Use `writeClient.patch(memberId).set(updates).commit()`
- Revalidate both paths (changes reflect in public directory)
- Return structured result

**Why 'server-only' import:**
writeClient contains SANITY_WRITE_TOKEN which must NEVER be exposed to browser. The 'server-only' package causes build error if accidentally imported in Client Component.

**Pattern reference:** Follow structure from `app/actions/auth.ts` (verifySession first, structured returns)
