<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete member directory system with two interfaces:
1. **Public directory** at /members - visitors browse approved members with photos and professional info
2. **Admin management** at /admin/members - admins review pending submissions and manage directory
3. **Dashboard stats** at /admin - real-time member counts and quick actions
  </what-built>

  <how-to-verify>
**Phase A: Public Directory Verification**

1. **Visit public member directory:**
   - Open http://localhost:3000/members in browser
   - Verify page title shows "Member Directory | Checkmate & Connect"
   - Check member count displays at top (e.g., "42 approved members")

2. **Test responsive layout:**
   - Desktop (wide browser): Verify 4 columns of member cards
   - Tablet (medium width): Verify 3 columns
   - Mobile (narrow width): Verify 2 columns
   - Test on physical mobile device if available

3. **Verify member card content:**
   - Each card shows member photo (not broken image icon)
   - Hover over card: photo transitions from grayscale to color
   - Hover overlay displays: member name, job title, company (if present)
   - LinkedIn link present (if member has LinkedIn profile)
   - Click LinkedIn link: opens in new tab to correct profile

4. **Check image optimization:**
   - Open browser DevTools → Network tab → Filter by "Img"
   - Reload /members page
   - Verify images load from Sanity CDN (cdn.sanity.io domain)
   - Check image URLs include parameters like `w=400&h=400&fit=crop&auto=format`
   - Verify images are WebP or AVIF format (efficient compression)

5. **Test home page integration:**
   - Visit http://localhost:3000 (home page)
   - Scroll to "Meet Our Community" section
   - Verify MemberHighlights shows real member photos (not placeholder stock images)
   - Click "See All Members" button → navigates to /members directory

**Phase B: Admin Management Verification**

6. **Access admin dashboard:**
   - Visit http://localhost:3000/admin
   - Verify dashboard shows three stat cards with real counts:
     - Pending Members (count of submissions awaiting review)
     - Total Members (count of approved members)
     - Recent Activity (count of approvals in last 7 days)
   - If pending members exist, verify "Pending Review" card displays

7. **Test member management table:**
   - Click "Pending Members" card or navigate to /admin/members
   - Verify default view shows pending members table
   - Check columns: Photo, Name/Title, Company, Submitted Date, Actions
   - Verify photos display as small rounded circles (not broken)

8. **Test filter tabs:**
   - Click "Approved" tab → URL changes to ?status=approved, table updates
   - Click "Rejected" tab → URL changes to ?status=rejected, table updates
   - Click "Pending" tab → returns to pending view
   - Use browser back button → filter state restored from URL

9. **Test approval workflow:**
   - Go to Pending tab (ensure at least one pending member exists)
   - Click "Approve" button on a pending member
   - Button shows "Approving..." during submission
   - After success, member disappears from pending table
   - Click "Approved" tab → approved member now appears in list
   - **Critical test:** Open /members in new browser tab → approved member immediately visible (no refresh needed)

10. **Test rejection workflow:**
    - Return to Pending tab
    - Click "Reject" button on another pending member
    - Member moves to Rejected tab
    - Verify rejected member does NOT appear in public /members directory

11. **Test delete action:**
    - Go to Rejected or Approved tab
    - Click "Delete" button on a member
    - Confirm member removed from table
    - If deleted member was approved, verify removed from public directory

**Phase C: Data Accuracy Verification**

12. **Verify member counts:**
    - Count members in /admin/members?status=pending
    - Check dashboard "Pending Members" stat card shows same count
    - Count members in /members public directory
    - Check dashboard "Total Members" stat card shows same count

13. **Verify privacy compliance:**
    - Open Sanity Studio at /studio
    - Find a member with status "pending" or "rejected"
    - Verify this member does NOT appear in /members public directory
    - Only approved members should be publicly visible

**Phase D: Mobile Responsiveness**

14. **Test on mobile device (if available):**
    - Visit /members on phone/tablet
    - Verify 2-column grid on phone, 3-column on tablet
    - Test member card hover (tap on mobile)
    - Check LinkedIn links work on mobile
    - Visit /admin/members on mobile
    - Verify admin table scrolls horizontally if needed

**Expected Behavior Summary:**
- Public directory only shows approved members (privacy)
- Member count is accurate and consistent
- Photos load quickly from Sanity CDN with optimization
- Approval workflow updates both admin and public views immediately
- Responsive grid adapts to screen size (2/3/4 columns)
- Admin actions (approve/reject/delete) work without errors
- Dashboard statistics reflect current database state
  </how-to-verify>
