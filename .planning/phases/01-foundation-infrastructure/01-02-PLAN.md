  <action>
1. Install dependencies:
```bash
npm install next-sanity @sanity/client @sanity/image-url @sanity/vision @portabletext/react sanity sanity-plugin-media
```

2. Update `lib/sanity/client.ts` with configured Sanity client:
```typescript
import { createClient } from '@sanity/client';

export const client = createClient({
  projectId: process.env.NEXT_PUBLIC_SANITY_PROJECT_ID!,
  dataset: process.env.NEXT_PUBLIC_SANITY_DATASET!,
  apiVersion: process.env.NEXT_PUBLIC_SANITY_API_VERSION || '2024-01-01',
  useCdn: true,
});
```

3. Update `lib/sanity/imageUrl.ts` with image URL builder:
```typescript
import imageUrlBuilder from '@sanity/image-url';
import { client } from './client';

const builder = imageUrlBuilder(client);

export function urlFor(source: any) {
  return builder.image(source);
}
```
This enables `urlFor(image).width(800).quality(75).format('avif').url()` pattern for FOUND-06.

4. Update `lib/sanity/queries.ts` with initial GROQ queries:
- `membersQuery`: Fetch published members ordered by name
- `blogPostsQuery`: Fetch published posts ordered by publishedAt desc
- `blogPostBySlugQuery`: Fetch single post by slug

5. Create `sanity.config.ts` at project root:
```typescript
import { defineConfig } from 'sanity';
import { structureTool } from 'sanity/structure';
import { visionTool } from '@sanity/vision';
import { schemas } from './lib/sanity/schemas';

export default defineConfig({
  projectId: process.env.NEXT_PUBLIC_SANITY_PROJECT_ID!,
  dataset: process.env.NEXT_PUBLIC_SANITY_DATASET!,
  basePath: '/studio',
  plugins: [structureTool(), visionTool()],
  schema: { types: schemas },
});
```

6. Create `app/studio/[[...index]]/page.tsx` with embedded Studio:
```typescript
'use client';
import { NextStudio } from 'next-sanity/studio';
import config from '@/sanity.config';

export const dynamic = 'force-static';

export default function StudioPage() {
  return <NextStudio config={config} />;
}
```
  </action>

Run `npm run dev` and navigate to `http://localhost:3000/studio`. If SANITY_PROJECT_ID is set in .env.local, Studio should load. If not set, verify the route exists and renders the Studio component (may show config error which is expected without credentials). Run `npx tsc --noEmit` to confirm types are clean.
  </verify>
  <done>
Sanity client configured with CDN. Image URL builder exports `urlFor` function. Studio route exists at /studio. GROQ queries defined for members and blog posts.

1. Create `lib/sanity/schemas/member.ts` using `defineType` and `defineField`:

Fields:
- `name` (string, required) -- Member's display name
- `slug` (slug, source: name) -- URL-friendly identifier
- `photo` (image, required, hotspot enabled) -- Profile photo (upload high-res, Sanity CDN optimizes per FOUND-06)
- `jobTitle` (string, required) -- Professional title
- `company` (string, optional) -- Company name
- `linkedIn` (url, optional) -- LinkedIn profile URL
- `bio` (text, optional) -- Short bio (max 300 chars)
- `status` (string, options: ['pending', 'approved', 'rejected'], default: 'pending') -- Approval status for FOUND-07 privacy opt-in
- `submittedAt` (datetime) -- When profile was submitted
- `approvedAt` (datetime) -- When profile was approved

Set preview to show name and photo.

2. Create `lib/sanity/schemas/blogPost.ts` using `defineType` and `defineField`:

Fields:
- `title` (string, required) -- Post title
- `slug` (slug, source: title, required) -- URL slug
- `coverImage` (image, hotspot enabled) -- Featured image
- `excerpt` (text, max 200 chars) -- Short description
- `body` (array of block + image) -- Portable Text rich content
- `publishedAt` (datetime) -- Publication date
- `author` (string) -- Author name

Set preview to show title and coverImage.

3. Update `lib/sanity/schemas/index.ts` to export both schemas:
```typescript
import member from './member';
import blogPost from './blogPost';

export const schemas = [member, blogPost];
```

  </verify>
