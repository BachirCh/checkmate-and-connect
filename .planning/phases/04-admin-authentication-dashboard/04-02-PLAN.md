  <name>Task 1: Create proxy.ts and admin login page</name>
  <files>
    proxy.ts
    app/admin/login/page.tsx

  <action>
Create `proxy.ts` at project root (Next.js 16 convention - NOT middleware.ts):
- Export async function `proxy(req: NextRequest)` (not default export)
- Use the existing `updateSession` from '@/lib/supabase/middleware' to refresh session and get supabaseResponse
- BEFORE calling updateSession, determine route type: isAdminRoute = path.startsWith('/admin'), isLoginRoute = path.startsWith('/admin/login')
- After updateSession, create a separate Supabase client inline (same pattern as middleware.ts) to call getUser() and check authentication
- If isAdminRoute AND NOT isLoginRoute AND no user: redirect to /admin/login
- If isLoginRoute AND user exists: redirect to /admin
- Otherwise return the supabaseResponse from updateSession
- Export config with matcher that excludes static assets: `/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)`
- IMPORTANT: proxy.ts runs on Node.js runtime in Next.js 16 (not Edge)

Create `app/admin/login/page.tsx` as a client component ('use client'):
- Use `useActionState` from 'react' (React 19) with the login action from '@/app/actions/auth'
- Form fields: email (type="email", required) and password (type="password", required)
- Show field-level errors from state.errors.email and state.errors.password
- Show general error from state.message
- Submit button with pending state (disable + show "Logging in...")
- Styling: Follow existing dark theme - min-h-screen bg-black, centered card with bg-gray-900 border border-gray-800 rounded-lg, max-w-md
- Input styling: bg-gray-800 border-gray-700 rounded-md, focus:ring-2 focus:ring-white (not blue - match black/white branding)
- Button: bg-white text-black hover:bg-gray-200 (match black/white branding, not blue)
  </action>
  <verify>
Run `npx tsc --noEmit`. Verify proxy.ts exists at project root. Verify login page renders without errors via `npm run build` or dev server.
  </verify>

proxy.ts protects /admin routes (redirects unauthenticated to /admin/login, redirects authenticated away from /admin/login). Login page shows centered dark-theme form with email/password fields, validation errors, and submit state.
  </done>
