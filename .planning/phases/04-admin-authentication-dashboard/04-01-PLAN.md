  <files>
    supabase/migrations/002_admin_roles.sql

  <action>
Create `supabase/migrations/002_admin_roles.sql` with:
1. `user_roles` table: id (bigint generated), user_id (uuid references auth.users on delete cascade, unique), role (text check in 'admin','user'), created_at (timestamptz default now())
2. Enable RLS on user_roles. Policy: admins can manage roles (auth.jwt()->>'role' = 'admin' for ALL operations)
3. `custom_access_token_hook` function (plpgsql, STABLE): reads role from user_roles for event user_id, sets claims.role (defaults to 'user' if no role found), returns modified event
4. Grant permissions: USAGE on schema public to supabase_auth_admin, SELECT on user_roles to supabase_auth_admin, EXECUTE on custom_access_token_hook to supabase_auth_admin
5. Revoke execute from authenticated and anon roles on the hook function (security)

Create `app/actions/auth.ts` with 'use server' directive:
- `login(prevState: any, formData: FormData)`: Validate email+password with Zod (email: z.string().email(), password: z.string().min(8)). On validation failure return {errors}. Call supabase.auth.signInWithPassword. On auth error return {message: 'Invalid email or password'}. After successful login, call supabase.auth.getUser() and check user.app_metadata?.role === 'admin'. If not admin, call supabase.auth.signOut() and return {message: 'Unauthorized: Admin access required'}. On success, redirect('/admin').
- `logout()`: Call supabase.auth.signOut(), then redirect('/admin/login').
- Import createClient from '@/lib/supabase/server'. Import redirect from 'next/navigation'. Import z from 'zod'.
- IMPORTANT: Call redirect() outside try/catch blocks (Next.js redirect throws internally).
  </action>
  <verify>
Run `npx tsc --noEmit` to verify TypeScript compilation passes. Confirm both files exist and have correct structure.
  </verify>
  <done>
SQL migration contains user_roles table, RLS policy, custom_access_token_hook function, and grants. Auth actions export login (with Zod validation + admin role check) and logout (with signOut + redirect).
  </done>
</task>
