**1. Create Sanity write client** (`lib/sanity/write-client.ts`):
- Import createClient from @sanity/client
- Configure with same projectId/dataset/apiVersion as read client
- Set `useCdn: false` (CRITICAL: writes must not use CDN)
- Set `token: process.env.SANITY_WRITE_TOKEN!` (server-only, NO NEXT_PUBLIC_ prefix)

**2. Create Zod validation schema** (`lib/validations/member-submission.ts`):
- Export `memberSubmissionSchema` with:
  - `name`: string, min 2 chars, max 100, required
  - `jobTitle`: string, min 2 chars, max 100, required
  - `company`: string, max 100, optional (allow empty string)
  - `linkedIn`: string, url(), regex `/linkedin\.com/` (flexible match, not strict path), optional or empty string
  - `photo`: instanceof File, refine max 5MB, refine type jpeg/png/webp
  - `recaptchaToken`: string, min 1
  - `_honey`: string, optional (honeypot)
- Export `MemberSubmissionData` type inferred from schema
- Create a separate `serverMemberSubmissionSchema` that uses z.any() for photo (since File won't be available as instanceof in Node server context — use refinements on size/type from FormData instead)

**3. Create Server Action** (`app/(public)/join/actions.ts`):
- `'use server'` directive at top
- Export `submitMemberAction(prevState: any, formData: FormData)` function
- Extract all fields from FormData (name, jobTitle, company, linkedIn, photo as File, recaptchaToken, _honey)
- Three-layer spam protection using existing `lib/spam-protection.ts`:
  1. `checkHoneypot()` — return error silently if triggered
  2. `rateLimit('member_submission', 10, 60000)` — 10 per minute
  3. `verifyRecaptcha(token, 'member_submission')` — check score >= 0.5
- Server-side validation with Zod (validate text fields; validate photo size <= 5MB and type is image/jpeg, image/png, or image/webp)
- Upload photo: `writeClient.assets.upload('image', photoFile, { filename, contentType })`
- Generate slug from name using simple slugify (lowercase, replace spaces with hyphens, remove special chars)
- Create member document: `writeClient.create({ _type: 'member', name, slug: { _type: 'slug', current: slug }, jobTitle, company, linkedIn, photo: { _type: 'image', asset: { _type: 'reference', _ref: imageAsset._id } }, status: 'pending', submittedAt: new Date().toISOString() })`
- On success: `redirect('/join/confirmation')`
- On validation error: return `{ success: false, error: string, fieldErrors: object }`
- On server error: return `{ success: false, error: 'Failed to submit. Please try again.' }`
- Wrap upload + create in try/catch

**4. Update next.config.ts**:
- Add `serverActions: { bodySizeLimit: '10mb' }` under experimental (needed for photo uploads, default 1mb is too small)

**Important:** Do NOT use `redirect()` inside try/catch — Next.js redirect throws an error internally. Call redirect AFTER the try/catch block, storing success state in a variable.
